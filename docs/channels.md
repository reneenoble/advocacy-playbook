# Promotion Channels

This page covers the channels available for promoting your samples, demos, and tools to developer audiences.

## Social Media

### X (Twitter) / Threads / Mastodon

Best for: Quick announcements, sharing links, starting conversations, and engaging in real time.

**Tips:**
- Keep messages concise and include a clear call to action.
- Use a short GIF or screenshot to boost engagement.
- Use relevant hashtags (e.g., `#GitHub`, `#Azure`, `#VSCode`, `#OpenSource`, `#DevTools`).
- Tag relevant accounts (team accounts, product accounts) to extend reach.
- Tweet threads are effective for step-by-step tips or summarizing a longer piece of content.

### LinkedIn

Best for: Reaching professional developers and decision-makers, sharing longer-form updates.

**Tips:**
- Write a brief personal narrative framing before sharing a link.
- Engage with comments promptly to boost algorithmic reach.
- Tagging people who contributed or are featured increases visibility.

### YouTube / Video

Best for: In-depth demos, walkthroughs, and tutorials.

**Tips:**
- Keep intro under 30 seconds — get to the demo quickly.
- Add chapters using timestamps in the description for long videos.
- Include a pinned comment with links to the repo and blog post.
- Optimise the title and description with keywords your audience searches for.

## Blogging

### Microsoft Tech Community

The official Microsoft tech blogging platform. Posts here receive promotion in Microsoft newsletters and social channels.

- Suitable for product-focused content, announcements, and in-depth tutorials.
- Recommended for content tied to GA releases or major feature launches.

### Dev.to / Hashnode / Medium

Community-first developer blogging platforms with strong organic reach.

- Great for more conversational, opinion-based, or tutorial content.
- Cross-posting from a personal blog is common and accepted.

### GitHub Blog

For content closely tied to GitHub products and open-source milestones.

- Coordinated with the GitHub communications team.
- High visibility for significant announcements.

### Personal Blogs

Advocates are encouraged to maintain personal blogs. Personal content often feels more authentic and can reach audiences outside of official channels.

## Microsoft Learn

[Microsoft Learn](https://learn.microsoft.com) is the primary platform for long-form educational content, including:

- **Learning Paths** – Multi-module guided learning journeys.
- **Modules** – Standalone interactive lessons.
- **Quickstarts & Tutorials** – Task-based documentation.

Publishing on Microsoft Learn increases discoverability through search engines and surfaces your content to developers actively learning.

## Events & Conferences

### Major Conferences

| Conference | Audience | Notes |
|---|---|---|
| Microsoft Build | .NET, Azure, cloud developers | Microsoft's largest developer conference |
| GitHub Universe | GitHub users, open source, DevOps | GitHub's flagship event |
| KubeCon / CloudNativeCon | Cloud-native, Kubernetes, infrastructure | CNCF flagship event |
| PyCon | Python developers | Large community conference |
| JSConf / NodeConf | JavaScript / Node.js developers | Web and server-side JS community |
| DockerCon | Container and platform developers | Container-focused audience |

### Meetups & Community Events

- Local developer meetups (meetup.com groups, Azure user groups, GitHub Satellite events)
- Hackathons and workshops
- Online community calls and office hours

**Tips for events:**
- Prioritise conferences where your target developer audience is most active.
- Submit talk proposals early — most CfPs open 3–6 months before the event.
- Follow up after events with a recap blog post or video.

## Newsletters & Aggregators

- **GitHub Changelog** – For shipping new features.
- **Microsoft Developer Newsletters** – Coordinate with the comms team to be included.
- **Hacker News / Reddit (r/programming, r/github, r/dotnet, etc.)** – Community-driven amplification; genuine content with real value tends to perform well.
- **Morning Brew, TLDR, Bytes.dev** – Developer-focused newsletters that often pick up interesting projects and tutorials.

---

## Open Source & GitHub

- **GitHub Stars / Topics** – A well-structured, high-quality repository with a great README attracts organic stars and discovery.
- **GitHub Discussions** – Engage directly with users of your samples and tools.
- **Trending Repositories** – Getting a sample project to trend on GitHub increases organic reach significantly.

---

## Choosing the Right Channel

| Goal | Recommended Channels |
|---|---|
| Quick announcement | X (Twitter), LinkedIn |
| Deep technical tutorial | Blog (Tech Community / Dev.to), Microsoft Learn |
| Demo / walkthrough | YouTube, conference talk |
| Community engagement | GitHub Discussions, Reddit, meetups |
| Long-term discoverability | Microsoft Learn, SEO-optimized blog, GitHub repo |
